四大公式
分類評估 // Precision：預測為正的有多少真的是正的（誤報率）
Precision = TP / (TP + FP)

// Recall：實際為正的有多少被找出來（漏報率）
Recall = TP / (TP + FN)

// Accuracy：整體答對率（類別不均衡時失效）
Accuracy = (TP + TN) / (TP + TN + FP + FN)

// F1 Score：Precision 與 Recall 的調和平均數
F1 = 2 × P × R / (P + R)
💡 調和平均數（Harmonic Mean）對低值更敏感：若 P=1.0, R=0.01，算術平均 ≈ 0.505，但 F1 ≈ 0.02，能正確反映模型很差的狀況。
🔢 真實考題逐步計算 ── 科三 Q16
📋 題目：Precision = 0.8，Recall = 0.6，求 F1 Score？
代入分子：
2 × P × R = 2 × 0.8 × 0.6 = 0.96
計算分母：
P + R = 0.8 + 0.6 = 1.4
相除得 F1：
F1 = 0.96 / 1.4 ≈ 0.686
注意：F1 介於 P(0.8) 和 R(0.6) 之間，且偏向較低的 Recall！
考試速記：Precision vs Recall 的取捨
場景優先重視原因
醫療診斷（癌症篩查）Recall 高寧可多報（FP），不能漏診（FN）
垃圾郵件過濾Precision 高不能把正常信誤標為垃圾（FP代價高）
法律文件搜尋Recall 高要確保不遺漏相關文件
推薦系統F1 平衡兩者
② PCA 特徵值計算 科三 Q35 直接計算！
PCA 核心公式
降維計算 // 各主成分解釋的變異比例
Explained Variance Ratio(PCi) = λi / Σλ

// 累積解釋變異量（決定保留幾個主成分）
Cumulative Variance(k) = Σk λi / Σλ
🔢 真實考題逐步計算 ── 科三 Q35
📋 題目：三個主成分特徵值 λ₁=6.0, λ₂=3.0, λ₃=1.0，保留能解釋 ≥80% 總變異量的主成分？
計算總特徵值：
Σλ = 6.0 + 3.0 + 1.0 = 10.0
各主成分解釋比例：
PC1: 6.0 / 10.0 = 60%
PC2: 3.0 / 10.0 = 30%
PC3: 1.0 / 10.0 = 10%
累積至超過 80%：
PC1 累積 = 60%（未達 80%）
PC1 + PC2 累積 = 60% + 30% = 90% ≥ 80% ✅
結論：保留前 2 個主成分即可，降至二維，保留 90% 資訊。
⚠️ 考試陷阱：
• B 選項說「僅保留第1主成分」→ 60% < 80%，不足！
• C 選項說「第3主成分貢獻30%不宜捨棄」→ 錯！PC3=10%，不是30%
• D 選項說「特徵值相差不大」→ 錯！6:3:1 差距明顯
③ 條件機率 & 貝氏定理 科三 Q22 / Q41
核心公式
機率計算 // 條件機率：在 B 發生的前提下，A 發生的機率
P(A|B) = P(A ∩ B) / P(B)

// 貝氏定理：由先驗機率推後驗機率
P(A|B) = P(B|A) × P(A) / P(B)

// Monte Carlo 模擬版本（科三 Q41）
P(A|B) ≈ A_and_B.sum() / B.sum()
🔢 真實考題 ── 科三 Q41：骰子條件機率
📋 題目：擲骰子，A=偶數（2,4,6），B=大於3（4,5,6）
用 Monte Carlo 程式碼求 P(A|B)，正確公式為？
理論計算（驗證）：
P(B) = P(大於3) = 3/6 = 0.5
P(A∩B) = P(偶數且大於3) = P(4,6) = 2/6 = 0.333
條件機率：
P(A|B) = P(A∩B)/P(B) = (2/6)/(3/6) = 2/3 ≈ 0.667
對應 Monte Carlo 程式碼：
A_and_B.sum() = 事件 A∩B 發生次數
B.sum() = 事件 B 發生次數
∴ P(A|B) = A_and_B.sum() / B.sum() → 答案 D
❌ 錯誤：A.sum()×B.sum() 在分母 → 計算的不是條件機率
🔢 真實考題 ── 科三 Q22：貝氏定理應用
📋 題目：電商平台預測顧客購買機率，使用貝氏定理，核心機制是？
// 貝氏分類器的後驗機率
P(購買 | 行為特徵) = P(行為特徵 | 購買) × P(購買) / P(行為特徵)

// 先驗機率 P(購買) = 歷史購買率（如 20%）
// 似然度 P(行為|購買) = 購買者的行為分佈
// 後驗機率 = 觀察到此行為後，購買的概率
✅ 答案 B：「以條件機率方式計算顧客屬於會購買/不會購買的分類機率」
──這正是貝氏定理的後驗機率計算。
④ 熵 & 資訊增益（決策樹）科三 Q8
公式定義
決策樹分裂 // Shannon 熵：不確定性的度量
H(S) = -Σ pᵢ × log₂(pᵢ)

// 資訊增益：分裂後不確定性的減少量
IG(S, A) = H(S) - Σ ( |Sᵥ| / |S| ) × H(Sᵥ)
🔢 案例計算：決策樹特徵選擇
📋 場景：10 筆資料，5 個買（+）5 個不買（−），考慮用「年齡」特徵分裂
父節點熵：
H(S) = -(5/10)×log₂(0.5) - (5/10)×log₂(0.5) = -(-0.5) - (-0.5) = 1.0 bit
（50/50 分佈 → 最大熵 = 1.0）
年齡特徵分裂後：
年輕（5人）：4買 1不買 → H = -(4/5)log₂(4/5)-(1/5)log₂(1/5) ≈ 0.722
年長（5人）：1買 4不買 → H = -(1/5)log₂(1/5)-(4/5)log₂(4/5) ≈ 0.722
資訊增益：
IG = 1.0 - (5/10×0.722 + 5/10×0.722) = 1.0 - 0.722 = 0.278 bit
選擇 IG 最大的特徵進行分裂！
p（正例比例）熵 H(p)意義
0 或 1（純節點）0 bit完全確定，無需再分裂
0.5（完全混亂）1.0 bit最不確定，最需要分裂
0.80.722 bit部分確定
⑤ 迴歸評估指標：MSE / RMSE / MAE / R² 科三 Q15 / Q38
四個公式對比
迴歸評估 // MAE：平均絕對誤差（對異常值不敏感）
MAE = (1/n) × Σ|yᵢ - ŷᵢ|

// MSE：均方誤差（懲罰大誤差）
MSE = (1/n) × Σ(yᵢ - ŷᵢ)²

// RMSE：均方根誤差（與 y 同單位，易解讀）
RMSE = √MSE

// R²：決定係數（解釋變異比例）
R² = 1 - SS_res/SS_tot = 1 - Σ(yᵢ-ŷᵢ)² / Σ(yᵢ-ȳ)²
🔢 案例計算：房價預測誤差
📋 4 筆資料：實際房價 vs 預測房價（萬元）
實際 yᵢ預測 ŷᵢ誤差 (y-ŷ)|y-ŷ|(y-ŷ)²
10009802020400
800820-2020400
1200115050502500
600630-3030900
MAE：
MAE = (20+20+50+30)/4 = 120/4 = 30 萬
MSE：
MSE = (400+400+2500+900)/4 = 4200/4 = 1050 萬²
RMSE：
RMSE = √1050 ≈ 32.4 萬（與房價同單位，可直接解讀）
💡 科三 Q15 考點：R² = 0.85 → 模型能解釋 85% 的目標變數變異（不是準確率85%！）
R² 值解讀模型品質
≥ 0.9模型解釋 ≥90% 變異極佳
0.7 ~ 0.9解釋 70~90% 變異良好
< 0.5解釋不到一半需改進
≤ 0比平均值預測還差模型無效
⑥ IoU / mAP（物件偵測）科一 Q7
IoU 公式
物件偵測 // IoU = 交集面積 / 聯集面積
IoU = Intersection / Union

// 判斷預測框是否正確（TP）的閾值
if IoU ≥ threshold: → True Positive
if IoU < threshold: → False Positive
🔢 IoU 計算案例
真實框（綠）：
x=[100,200], y=[80,180] → 面積 = 100×100 = 10000
預測框（藍）：
x=[130,240], y=[100,200] → 面積 = 110×100 = 11000
計算交集：
x重疊=[130,200]=70，y重疊=[100,180]=80
交集面積 = 70×80 = 5600
計算聯集：
聯集 = 10000 + 11000 - 5600 = 15400
IoU = 5600 / 15400 ≈ 0.364
若閾值=0.5，此預測框為 FP（未達標準）
💡 科一 Q7 考點：IoU 閾值越高 → 判定標準越嚴格 → 模型偵測越精準（A正確）
高閾值 ≠ Precision降低（C錯誤）！高閾值通常使 Recall 降低（更多框被排除）
⑦ Softmax 函數 科一 Q8
公式與特性
分類輸出層 // Softmax：將原始分數轉為機率分佈
σ(zᵢ) = exp(zᵢ) / Σ exp(zⱼ)
🔢 計算案例
📋 模型輸出：z = [2.0, 1.0, 0.1]（三個類別的 logit 分數）
計算各類別的 exp：
exp(2.0) = 7.389
exp(1.0) = 2.718
exp(0.1) = 1.105
總和 = 7.389 + 2.718 + 1.105 = 11.212
轉換為機率：
P(類別1) = 7.389/11.212 ≈ 0.659（65.9%）
P(類別2) = 2.718/11.212 ≈ 0.242（24.2%）
P(類別3) = 1.105/11.212 ≈ 0.099（9.9%）
總和 = 1.000 ✅（所有輸入資訊都保留）
✅ 科一 Q8 答案 C：「Softmax 會保留所有輸入資訊，但以比例表示」
──每個類別的分數都被轉換為機率，沒有資訊被丟棄。
❌ Max-Pooling 只保留最大值，丟棄其他資訊，兩者性質截然不同！
⑧ L1 / L2 正則化 科三 Q2 / Q32
公式比較
正則化 // L1 Lasso：加入參數絕對值懲罰 → 產生稀疏模型
Loss_L1 = Loss + λ × Σ|wᵢ|

// L2 Ridge：加入參數平方懲罰 → 縮小參數值
Loss_L2 = Loss + λ × Σwᵢ²

// Elastic Net：L1 + L2 混合
Loss_EN = Loss + λ₁×Σ|wᵢ| + λ₂×Σwᵢ²
L1（Lasso）L2（Ridge）
懲罰形式|w|（絕對值）w²（平方）
效果使部分 w → 0（稀疏）使所有 w 變小
特徵選擇✅ 自動特徵選擇❌ 保留所有特徵
解釋性高（剩餘特徵少）中
適用場景高維稀疏特徵、特徵選擇特徵共線性問題
考試關鍵字「稀疏」「壓縮為零」「特徵選擇」「縮小」「共線性」「穩定」
💡 科三 Q2 考點：L1（Lasso）→「產生稀疏模型，使部分參數收斂為零」= 自動特徵選擇
💡 科三 Q32 考點：特徵有多重共線性 → 用 L1 Lasso 讓不重要特徵係數縮為零
